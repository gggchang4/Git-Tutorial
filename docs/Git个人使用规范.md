# Git个人使用规范

## 文档定位

本手册用于沉淀个人项目和小型协作中的 Git 使用规范，重点是形成一套可执行、可长期坚持、适合维护的轻量化规则。

## 规范目标

- 统一分支命名和分支职责
- 统一提交信息格式和提交粒度
- 统一单人开发与小型协作流程
- 减少混乱提交、无效分支和不可追踪修改

## 章节规划

### 1. 规范说明

- 规范适用范围
- 硬性约束、推荐做法、可选优化的区分
- 公开依据与个人约定的区分

### 2. 分支管理规范

- `main`
- `develop`
- `feature/*`
- `bugfix/*`
- `hotfix/*`
- `temp/*`

### 3. 提交规范

- 提交信息结构
- 提交类型枚举
- 英文提交要求
- 小步提交原则
- 提交前检查项

### 4. 单人开发流程

- 日常开发流程
- 合并与发布流程
- 标签策略

### 5. 小型协作流程

- 分支创建
- 每日同步
- PR / MR 流程
- 审核与合并

### 6. 冲突处理规范

- 冲突前沟通
- 冲突解决步骤
- 解决后验证

### 7. 配套规范

- `.gitignore`
- 敏感信息管理
- 测试与提交前检查
- 版本发布说明

## 写作要求

- 所有规范必须尽量写成可以执行的动作
- 涉及大厂实践时必须标明公开来源
- 与正文手册重复的命令解释应保持简洁，必要时链接到其他 Part
